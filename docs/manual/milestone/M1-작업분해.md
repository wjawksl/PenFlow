# 네이버 블로그 AI 작성 — M1 상세 작업 분해 (WBS)

> 기준 문서: `../02-기능설계서.md`, `../03-구현체크리스트.md`
> M1 = **Walking Skeleton** (최소 동작 관통). 범위: 공통기반(최소) + ① + ③ + ⑤ + ⑥ + ⑦

---

## M1의 핵심 개념: "가장 얇은 수직 슬라이스"

M1의 목적은 기능을 다 만드는 게 아니라, **전 구간을 한 번 관통시켜 가장 위험한 통합 지점을 초기에 검증**하는 것이다. 각 컴포넌트는 "최소한만" 구현하고, 부가 기능(이미지·표·자동화 등)은 의도적으로 제외한다.

```
[목표 시나리오]
 키 1개 저장 → 주제 1개 입력 → AI가 텍스트 본문 생성 → 편집기에 삽입 → 임시저장
```

가장 위험한 지점은 **③생성 → ⑥삽입 → ⑦저장**의 연결부(외부 편집기 자동 조작)이므로, 여기를 M1에서 먼저 뚫는다.

## 작업 분해 (WBS)

### WP0. 공통 기반 (최소만)
| ID | 작업 | 산출물 | DoD |
|---|---|---|---|
| 0-1 | 마커 규약 정의 (M1은 **소제목 마커만**) | 마커 상수 1곳 | 생성기·삽입엔진이 같은 상수 참조 |
| 0-2 | 데이터 모델 최소형 | `Settings(key,model)`, `Topic`, `Payload(contentHtml, publishOption)` | 타입 정의 완료 |
| 0-3 | 로컬 저장소 read/write 래퍼 | 저장 인터페이스 | 저장→재기동→인출 동작 |
| 0-4 | 진행/에러 통지 채널 | 공통 함수 | 1곳에서 호출되면 UI+로그 출력 |

> **제외**: 이미지/표/백링크 마커, AutoPostSession, Visual 모델

### WP1. ① 설정 관리자 (최소)
| ID | 작업 | DoD |
|---|---|---|
| 1-1 | AI 키 **1개** 입력·저장·인출 | 저장 후 재조회되면 통과 |
| 1-2 | 모델 선택·저장 | 선택값 유지 |
| 1-3 | 키 미설정 시 생성 차단 + 안내 | 키 없으면 "키를 먼저 등록" 메시지 |

> **제외(→M5/M2)**: 복수 키 순환(R-0.2), 내보내기/불러오기(R-0.4), 시계 오차 처리(R-0.5)

### WP2. ③ 본문 생성기 (직접 호출만)
| ID | 작업 | DoD |
|---|---|---|
| 2-1 | 주제+프롬프트 → 지시문 조립 | 조립 문자열 생성 |
| 2-2 | **직접 호출 방식**만 구현 (방식 B 제외) | AI 응답 수신 |

> **제외**: 웹 자동화 방식(B), 참고자료 크롤링, 프롬프트 저장/관리, 일괄 자동생성

| ID | 작업 | DoD |

### WP4. ⑥ 삽입 엔진 ⭐ (M1 최대 난관)
| ID | 작업 | DoD |
|---|---|---|
| 4-1 | 편집기 영역 **폴링+타임아웃** 탐색 | 로드 지연돼도 잡아냄 |
| 4-2 | 초기 팝업 정리 | 방해 팝업 자동 닫힘 |

> **제외**: 발행(publish) 경로 — M1은 임시저장만

## 의존성 & 작업 순서

```
0-1·0-2·0-3·0-4 (기반)
      │
      ▼
   1-1·1-2·1-3 (키)
      │
      ▼
   2-1~2-4 (생성)  ──► 3-1 (전달) ──► 4-1~4-6 (삽입) ──► 5-1·5-2 (저장)
                                         ▲
                              ⭐ 여기를 가장 먼저 PoC 권장
```

> **권장 진행 팁**: 4번(삽입엔진)이 가장 불확실하므로, 2번(생성)을 기다리지 말고 **하드코딩한 더미 본문**으로 4번을 먼저 검증(PoC)한다. 외부 편집기 조작이 되는지부터 확인한 뒤 생성기를 연결하면 리스크가 크게 줄어든다.

## M1 완료 판정 (통합 DoD)
- [ ] 키 1개 저장한 상태에서
- [ ] 주제 1개 입력 → 텍스트 본문 생성(소제목 마커 포함) →
- [ ] 편집기에 제목+본문이 순서대로 자동 입력되고 →
- [ ] 임시저장까지 **사람 개입 없이** 완료
- [ ] 중간 1개 단계 실패 시 패닉 없이 사유 안내 후 종료
