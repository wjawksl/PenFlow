# 펜플로우 — M3 상세 작업 분해 (WBS)

---

## M3의 핵심 개념: "본문을 다듬고 그림을 입힌다"

M1=관통, M2=부가요소 합성. M3는 **품질**을 올린다 — 본문을 검증·정리(⑩)하고 이미지·썸네일을 만든다(⑨). MV3 서비스 워커엔 DOM/Canvas 가 없으므로(05 §1) **Offscreen 문서**를 처음 도입해 변환·이미지 처리를 그곳에서 돌린다.

```
[M1] 관통   [M2] 부가요소 합성   [M3] 본문 검증·정리(⑩) + 비주얼(⑨)  ← 품질
```

> **사용자 체감 포인트**: M2까지는 AI가 뱉은 마크다운(표·리스트 등)이 평문으로 깨져 들어갔다. **WP1 변환(turndown/marked)** 이 이를 정식 HTML 로 정리한다 — 가장 먼저 체감되는 개선.

## 의존성 & 작업 순서

```
WP0 Offscreen 도입
   │
   ├─► WP1 변환(turndown/marked) ⭐ 먼저 — 표 깨짐 해결, 외부키 불필요
   │      └─► WP3 찾기·바꾸기
   ├─► WP2 형태소(Kiwi WASM)
   └─► WP4 비주얼 → WP5 중복회피·압축 → WP6 모델일관성
                          │
                          └─► WP7 정합성 → WP8 삽입(이미지)
```

> **권장 착수**: WP1 변환부터. 외부 키·이미지 불필요, 사용자 체감 큼(표/리스트 정리). Offscreen(WP0) 없이 **UI 페이지에서 변환 PoC** 후 Offscreen 일원화해도 됨(05 §10 열린 항목).
