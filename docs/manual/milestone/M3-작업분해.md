# 펜플로우 — M3 상세 작업 분해 (WBS)

---

## M3의 핵심 개념: "본문을 다듬고 그림을 입힌다"

M1=관통, M2=부가요소 합성. M3는 **품질**을 올린다 — 본문을 검증·정리(⑩)하고 이미지·썸네일을 만든다(⑨). MV3 서비스 워커엔 DOM/Canvas 가 없으므로(05 §1) **Offscreen 문서**를 처음 도입해 변환·이미지 처리를 그곳에서 돌린다.

```
[M1] 관통   [M2] 부가요소 합성   [M3] 본문 검증·정리(⑩) + 비주얼(⑨)  ← 품질
```

> **사용자 체감 포인트**: M2까지는 AI가 뱉은 마크다운(표·리스트 등)이 평문으로 깨져 들어갔다. **WP1 변환(turndown/marked)** 이 이를 정식 HTML 로 정리한다 — 가장 먼저 체감되는 개선.

### WP2. ⑩ 형태소 분석 — 키워드 밀도 (Kiwi WASM)
| ID | 작업 | DoD |
|---|---|---|
| 2-1 | Kiwi WASM 번들·로드(Offscreen 또는 UI) | 토큰화 동작 |
| 2-2 | 키워드 출현 횟수·밀도 집계 → `DensityReport` | 표 출력 |
| 2-3 | 권장 밀도 범위 **설정 가능** + 벗어난 키워드 경고 (R-8.2) | 경고 표시 |

> **구현 현황 (2026-06-06)**: ✅ 완료 — **단, Kiwi WASM 은 보류하고 경량 카운트로 구현**.
> - **스파이크 결론(Kiwi 보류)**: kiwi-nlp 0.23.0 wasm(3.6MB)은 node 에서 토큰화 정상 동작 확인("아이폰15"→`아이폰/NNP 15/SN`, 조사 `이/JKS` 분리). 그러나 **모델이 필수**다: npm 패키지엔 모델 미포함, Kiwi 릴리스 base 모델은 `cong.mdl` 75MB 를 포함해 **최소 기능 세트 ≈ 94MB(압축해제)**. 옵션 사전(default/multi/typo)을 다 꺼도, cong.mdl 없이는 build 실패(검증함). MV3 확장 번들엔 비현실적이고 원격+캐시도 첫 다운로드 ~80MB. 밀도검사 1개 기능 대비 과해 **보류**.
> - **경량 구현**: `convert/density.ts` — 마커 strip + 태그/엔티티 제거(`extractText`, DOM 불필요 → SW 가능) → 공백 토큰 분모 + 키워드 정규화 부분일치 카운트(`countKeyword`, 대소문자 무시·조사결합 포착) → `analyzeDensity` 가 `DensityReport`(횟수·밀도·verdict) 생성. 타깃 키워드는 이미 아는 명사(topic.keyword+연관)라 형태소 없이도 동일 카운트(스파이크에서 "아이폰" 2회 일치 확인).
> - **R-8.2 설정 가능**: `Settings.densityRange`(기본 1~3%) + `setDensityRange()`. 사이드패널에서 min~max 편집·저장(다음 세션 유지), 메인 키워드 자동 포함 + 추가 키워드(쉼표) 입력, 횟수·밀도·판정(✅적정/⚠과다/△부족) 표.
> - background `density.analyze` 핸들러(저장 본문 집계). `tests/density.test.ts` 8케이스. **추후 정확도가 필요하면** `countKeyword`/토큰화 자리에 Kiwi(Offscreen 로드)를 끼울 수 있게 함수 경계 단순 유지.
