# 펜플로우 — M2 스파이크 (SmartEditor 표/링크 삽입 실측)

> 기준: `M2-작업분해.md` §스파이크, `../07-editor-automation-spec.md` §6(SHOP/PRODUCT 주입).
> 목적: ④ 합성기가 만든 표·링크 InsertTask 를 ⑥ 삽입엔진이 **어떻게 넣어야 하는지** 확정.
> ⚠️ 실제 브라우저 + 네이버 로그인 환경에서만 가능(헤드리스 불가).

---

## 1. 실행 방법

1. 네이버 블로그 **글쓰기**(팝업 모드) 열고 본문 보이는 상태. `F12` → Console.
2. `tools/spike/wp6-table-link-probe.js` 전체 붙여넣기 → **툴바 버튼 정찰**(읽기전용) 출력.
3. 표 paste 실측: `PENFLOW_WP6 = 'table'` 실행 후 프로브 재실행.
   링크 paste 실측: `PENFLOW_WP6 = 'link'` 실행 후 재실행.
   (본문에 테스트 내용 삽입됨 → 확인 후 `Ctrl+Z`)
4. 출력 JSON 전체 복사해 전달. (`window.__PENFLOW_WP6__` 에도 보관)

---

## 2. 미지수 체크리스트 (실측 후 채움)

| # | 미지수 | 측정 | 결과 (2026-06-04 실측) |
|---|---|---|---|
| 1 | HTML `<table>` paste → SE 표 컴포넌트(.se-table) 변환? | `PENFLOW_WP6='table'` | ✅ **변환됨**. `.se-table +1`, class `se-component se-table se-l-default`. HTML paste 가능 |
| 2 | 변환 안 되면 툴바 표 버튼 필요 | — | 불필요(paste 로 됨) |
| 3 | 링크 paste → SE og/링크 카드? | `PENFLOW_WP6='link'` | ✅ **인라인 링크** `.se-link +1` (class `se-ff-nanumgothic se-fs15 se-link __se-node`). og카드(.se-oglink) 아님 |
| 4 | 링크 후 자동 생성 불필요 텍스트(07 §6 SHOP) | DOM 관찰 | ✅ **없음**. 인라인 링크라 부가 DOM 생성 안 됨 → 제거 불필요 |
| 5 | 표/링크 버튼 셀렉터 | — | 불필요 |

---

## 3. 실측 결과 (코드 반영 완료)

```
표 삽입 기법    : ✅ HTML <table> paste → .se-table 컴포넌트 자동 변환
링크 삽입 기법  : ✅ HTML <a href> paste → .se-link 인라인 링크 (og카드 아님)
링크 부가텍스트 : 없음 (제거 불필요)
결론           : 표·링크 모두 pasteHtml 재사용. 툴바 버튼 조작 불필요.
```

> 코드 반영: `engine.ts` `taskToHtml()` + InsertQueue 순차 paste 실행, `compose()` 를 background generate 에 배선, `Payload.insertQueue` 추가.
