# 펜플로우 — M1 스파이크 (네이버 SmartEditor 실측)

> 기준: `../07-editor-automation-spec.md` §12 체크리스트.
> 목적: 코딩된 ⑥ 삽입 엔진의 **예시 셀렉터·주입 기법**을 실제 네이버 SmartEditor 값으로 교체하기 위한 실측.
> ⚠️ 이 측정은 **실제 브라우저 + 네이버 로그인** 환경에서만 가능하다(헤드리스/CI 불가).

---

## 1. 실행 방법

1. 크롬/웨일에서 네이버 블로그 **글쓰기**를 연다. 권장: **팝업 모드**(R-4.5). 본문 입력칸이 보이는 상태.
2. `F12` → **Console** 탭.
3. `tools/spike/smarteditor-probe.js` 전체를 복사해 콘솔에 붙여넣고 Enter.
4. 출력된 표 + 맨 아래 **JSON 전체**를 복사해 전달한다. (`window.__PENFLOW_SPIKE__` 에도 보관됨)
5. (선택) paste 주입까지 보려면 먼저 콘솔에 `PENFLOW_PROBE_PASTE = true` 실행 후 프로브 재실행. 본문에 `[펜플로우 paste 테스트]`가 들어가면 paste 기법이 먹히는 것 → `Ctrl+Z`로 되돌린다.

## 2. §12 미지수 체크리스트 (실측 후 채움)

| # | 미지수 | 측정 방법 | 결과 (2026-06-04 실측) |
|---|---|---|---|
| 1 | 본문이 iframe 인가, 같은 문서 contenteditable 인가? | 프로브 `frames` 출력 | ✅ **iframe[name=mainFrame] 안**. top 은 contenteditable 0개, iframe 에 1개 |
| 2 | 격리 월드로 접근 가능? `world:MAIN` 필요? | iframe `accessible:true` 여부 | ✅ **same-origin → 접근 가능. world:MAIN 불필요**. top CS 에서 `iframe.contentDocument` 직접 조작 |
| 3 | 텍스트/HTML paste 가 먹히는가? sanitize 동작? | `PENFLOW_PROBE_PASTE=true` | ✅ **먹힘**. 합성 `ClipboardEvent('paste')` 를 에디터가 가로채 처리(`eventCancelled:true` = preventDefault) → 본문에 삽입됨. `pasteHtml` 1차 기법(DataTransfer paste dispatch) 그대로 동작 |
| 4 | 이미지 삽입 경로와 완료 감지 신호 | (M3 범위) | — |
| 5 | 제목 필드 안정 셀렉터 | 프로브 `titleField` | ✅ `.se-title-text` (`.se-documentTitle` 은 버튼 포함 래퍼 → 부적합) |
| 6 | 저장/발행/확인 버튼 안정 셀렉터 | 프로브 버튼 매칭 | ✅ 저장 `.save_btn__bzc5B`, 발행 `.publish_btn__m9KHH`. **해시 클래스 동적** → `class*="save_btn"` 부분매칭. 발행 확인 버튼은 미등장(발행 클릭 후) |
| 7 | 서식 적용 UI 조작법 | 툴바 DOM 관찰 | ⬜ 미측정 (M1 은 인라인 스타일 best-effort) |
| 8 | 초기 팝업 종류·닫기 셀렉터 | 프로브 `initialPopupClose` | ⬜ 측정 시점 팝업 없음(매칭 0). '이어쓰기' 등은 재로그인/임시글 상태에서 재확인 |
| 9 | 팝업/사이드바 모드 차이 | 두 모드 비교 | ⬜ 팝업 모드만 측정(`mode: wide`). 사이드바 미측정 |

## 3. 실측 결과 (selectors.ts 반영 완료)

```
editorReady     : .se-canvas
titleField      : .se-title-text
bodyArea        : .se-content [contenteditable="true"]  /  div[contenteditable="true"]
initialPopupClose: (측정 시 없음 — 폴백 유지)
saveDraft       : button[class*="save_btn"]   (실측 .save_btn__bzc5B, 해시 동적)
publishOpen     : button[class*="publish_btn"] (실측 .publish_btn__m9KHH)
publishConfirm  : 미측정 (발행 M2+)
본문 위치        : iframe[name=mainFrame]  (same-origin)
주입 기법        : ✅ paste (합성 ClipboardEvent → 에디터가 가로채 처리, 검증됨)
world:MAIN 필요  : 아니오 (same-origin iframe 직접 접근)
```

> 코드 반영: `src/lib/selectors.ts`(실측 셀렉터), `src/components/insert/dom.ts`(`findEditorDoc()` iframe 타깃), `engine.ts`·`publish/index.ts`(editor 문서 기준 조작).
