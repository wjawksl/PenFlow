## M2의 핵심 개념: "본문 앞뒤를 채운다"

M1이 *주제→텍스트→삽입*의 가장 얇은 관통이었다면, M2는 그 앞단(**주제를 어떻게 고르나** = ②)과 뒷단(**본문 마커 자리에 무엇을 끼우나** = ④)을 채운다. 이미지·검증은 아직 제외(M3). 즉 M2 종료 시 "키워드로 주제를 발굴하고, 링크·상품표·광고문구가 박힌 글을 삽입"할 수 있다.

```
[M1] 주제(수동입력) → 텍스트 → 삽입
[M2] 주제(A/B/C 발굴) → 텍스트 + (링크·표·CTA·백링크·광고문구) 합성 → 삽입
```

가장 위험한 지점은 **④ 합성의 정합성**(마커 순서·개수, 광고문구 누락 금지)과 **검색광고 API 서명**(HMAC + 시계 오차)이다.

## 의존성 & 작업 순서

```
WP0 (마커활성·정합성·표컴포넌트)
   │
   ├─► ② WP1·2·3·4 (주제 3경로)  ── 본문 생성(M1 ③) 입력
   │
   └─► ④ WP5 (합성) ──► ⑥ WP6 (삽입 확장)
                            ▲
                  ⭐ 6-4 SE 표/링크 삽입 스파이크를 먼저 PoC
```

> **권장 팁**: M1과 동일하게 외부 편집기 의존부(6-1 표 삽입)가 가장 불확실하다. **더미 InsertQueue(표·링크 포함)로 6번 삽입을 먼저 검증**한 뒤 ④ 합성을 연결한다. 검색광고 서명(1-2)도 인증·시계 이슈가 있어 조기 PoC 권장.
