# 펜플로우 (PenFlow) — UI 설계서 (Figma 제작용)

> 기준 문서: `01-scenario.md`(흐름), `02-functional-design.md`(컴포넌트 ①~⑩), `04·05`(Side Panel·Options 표면), `08-interface-data-schema.md`(화면에 뜨는 필드)
> 목적: 이 문서만 보고 **피그마로 화면을 그릴 수 있도록** 화면 구조·구성요소·상태·문안·치수를 확정한다.
> 역할 분담: **구조·구성·상태·한글 문안·치수 = 본 문서가 확정(고정).** **색·타이포·아이콘·일러스트·무드 = 디자이너(나노바나나) 재량(자유).** 이 문서는 와이어프레임 수준이며, 시각 스타일은 의도적으로 비워 둔다.

---

## 1. 디자인 컨텍스트 & 제약 (가장 먼저 읽을 것)

- **두 개의 표면**이 있고 폭이 다르다.
  - **Side Panel (주 UI)**: 브라우저 우측에 붙는 **좁고 긴 세로 패널**. 설계 기준폭 **360px**(최소 320 / 여유 400), 높이는 가변·세로 스크롤. 단일 작성·비주얼·검토·발행·자동 포스팅이 모두 여기.
  - **Options (설정 페이지)**: 일반 웹페이지 레이아웃. 콘텐츠 기준폭 **720px**(중앙 정렬, 최대 960). 키·옵션 설정 전용(①).
- **모바일 아님.** 데스크톱 브라우저 안의 패널이다. 단, 좁은 폭이라 모바일 세로 화면처럼 **1열 스택**으로 설계한다.
- **언어: 한국어.** 모든 문안은 본 문서의 한글을 그대로 사용.
- **마커는 사용자에게 보이지 않는다.** 본문 미리보기에서 `[[PF:...]]`는 제거된 상태로 보여준다(06).
- **편집기 조작 중에는 네이버 글쓰기 탭이 따로 떠 있다.** 펜플로우 패널은 그 옆에서 진행 상태를 보여주는 역할(05).

---

## 2. 레이아웃 시스템

- **간격 스케일**: 4 / 8 / 12 / 16 / 24 / 32 px (8 기준).
- **터치/클릭 타깃**: 최소 높이 36px(버튼·행).
- **패널 내부 여백**: 좌우 16px, 섹션 간 24px.
- **1열 스택**: 모든 입력·표는 가로 꽉 채움(full-bleed minus 여백). 2열은 폭 부족으로 지양.
- **고정 영역 + 스크롤 영역**: 헤더(상단 고정) / 콘텐츠(스크롤) / 진행·액션 바(하단 고정).
- 시각 토큰(색/폰트/라운드/그림자)은 **디자이너가 정의** — 본 문서는 위계만 표기(예: 주요 버튼/보조 버튼/위험 버튼).

## 3. 정보구조(IA) & 내비게이션  ※ 제안 — 조정 가능

```
펜플로우 (Side Panel)
├── [모드 전환]  단일 작성  |  자동 포스팅
├── 단일 작성 = 5단계 스텝
│     ① 주제 → ② 작성 → ③ 비주얼 → ④ 검토 → ⑤ 삽입·발행
│     (상단 스텝 인디케이터 + 하단 '이전/다음')
├── 자동 포스팅 (목록·예약·간격·진행)
└── ⚙ 설정  → Options 페이지로 이동(별도 탭)
```

- 좁은 폭이라 **병렬 탭보다 스텝 위저드**가 적합(한 번에 한 화면). 단계 건너뛰기 허용(비주얼·검토는 선택, R-7.1·R-8.1).
- 상단 스텝 인디케이터는 5개 점/번호 + 현재 단계 라벨.

## 4. 공통 컴포넌트 라이브러리

각 컴포넌트는 **상태(state)** 를 모두 그려야 한다.

| 컴포넌트 | 변형/상태 |
|---|---|
| 버튼 | 주요 / 보조 / 위험 / 텍스트 · 상태: 기본·hover·press·disabled·**로딩(스피너+라벨)** |
| 입력 필드 | 단일행 / 멀티라인 · 상태: 기본·포커스·에러(빨강 보더+헬프텍스트)·disabled |
| 드롭다운(Select) | 기본·열림·선택됨·disabled |
| 토글 / 체크박스 | on / off / disabled |
| 슬라이더 | 값 라벨 동반(예: 품질 80%) |
| 세그먼트 탭 | 2~3개 항목, 선택 강조 |
| 표(Table) | 헤더 + 행 · 상태: 데이터·빈(empty)·로딩(스켈레톤) · 행 선택/클릭 |
| 이미지 카드 | 썸네일 + 액션(삭제·다운로드) · 선택 상태 · 빈 슬롯 |
| 진행 바 | 단계 진행률(%) + 현재 작업 텍스트 |
| 토스트 | 성공 / 경고 / 오류 (자동 사라짐) |
| 배너(인라인 알림) | 정보 / 경고 / 오류 (상시 노출, 예: 시계 오차 경고) |
| 모달 | 확인/취소 (예: 발행 확인, 일괄 삭제) |
| 칩/태그 | 키워드·소스 on/off |
| 드래그앤드롭 영역 | 기본 / 드래그오버 / 업로드중 |

---

## 5. 공통 상태 패턴 (모든 화면 공통으로 그릴 것)

- **로딩**: 스켈레톤(표/카드) 또는 버튼 내 스피너.
- **빈 상태**: 일러스트(자유) + 한 줄 안내 + 1차 액션 버튼.
- **에러**: 인라인 배너(상단) 또는 토스트 + 재시도 버튼. 문안은 §10.
- **진행 중(장시간)**: 하단 진행 바 + 단계 텍스트. 중단 버튼 노출.
- **성공**: 토스트 + 다음 단계로 자동/수동 이동.

## 6. 화면별 명세

> 표기: `[버튼]` 버튼, `(입력)` 입력칸, `▼` 드롭다운, `◉/○` 토글, `▢` 체크박스.

### S0 — 패널 셸 (모든 화면 공통 프레임)

```
┌──────────────────────────────┐  ← 상단 고정
│ 펜플로우      [단일|자동]  ⚙  │  헤더: 로고 · 모드세그먼트 · 설정
├──────────────────────────────┤
│ ①─②─③─④─⑤   "② 작성"       │  스텝 인디케이터(단일 모드일 때)
├──────────────────────────────┤
│                              │
│      (스크롤 콘텐츠 영역)      │
│                              │
├──────────────────────────────┤  ← 하단 고정
│ ▓▓▓▓▒▒▒▒ 62%  본문 생성 중…  │  진행 바(작업 중일 때만)
│ [이전]              [다음]   │  내비 액션
└──────────────────────────────┘
```

- 헤더는 모든 화면 공통. 모드 세그먼트로 단일/자동 전환. ⚙는 Options 새 탭.
- 진행 바는 장시간 작업 중에만 등장. 평소엔 숨김(높이 0).
- 상태 점(헤더 우측, 선택): AI 키 정상/소진, 시계 오차 등 전역 경고 표시.

### S1 — 설정 (Options 페이지) · 컴포넌트 ①

폭 720 기준, 카드형 섹션 세로 나열.

```
┌───────────────────────────────────────────┐
│ 설정                          [내보내기 ▼] │  파일/클립보드/클라우드
├───────────────────────────────────────────┤
│ ▣ AI 본문 생성 키 (필수)                    │
│   ┌ 키 #1  (••••••••)            [삭제]    │  복수 등록(R-0.1)
│   ├ 키 #2  (••••••••)            [삭제]    │  소진 시 자동 순환(R-0.2)
│   └ [+ 키 추가]                            │
│   모델  [ gemini-… ▼ ]          [저장]     │
├───────────────────────────────────────────┤
│ ▣ 주제 분석 키 (선택)                       │
│   검색광고  (API Key)(Secret)(CUSTOMER_ID) │
│   검색      (Client ID)(Secret)            │
├───────────────────────────────────────────┤
│ ▣ 이미지·제휴 키 (선택)                     │
│   이미지 소스 (Key) / 제휴 (Access)(Secret)│
├───────────────────────────────────────────┤
│ ▣ 본문 서식 기본값                          │
│   줄간격[▼] 글꼴[▼] 글자크기[▼]            │
└───────────────────────────────────────────┘
```

- 상시 배너(보안): "모든 키는 이 PC에만 저장되며 외부로 전송되지 않습니다."(R-0.3)
- 조건부 경고 배너: "PC 시계가 정확해야 검색광고 인증이 동작합니다."(R-0.5)
- 저장 성공/실패 → 토스트.
- 상태: 키 0개(빈), 1개 이상, 저장 중(버튼 로딩).

### S2 — 주제 선정 · 컴포넌트 ②  (세그먼트로 A/B/C 전환)

```
┌──────────────────────────────┐
│ [ 키워드 | 경쟁블로그 | 연관 ] │  세그먼트 탭 = 경로 A/B/C
├──────────────────────────────┤
│ ─ 경로 A (키워드) ─           │
│ (메인 키워드            )      │
│ [연관 키워드 찾기]            │
│ ┌ 키워드 │ 검색량 │ 경쟁도 ┐  │  표(검색량·경쟁도는 선택)
│ │ …………  │  1,200 │  낮음  │  │  행 클릭 → 주제 선택
│ └─────────────────────────┘  │
│ [목록 업로드] [목록 다운로드]  │
└──────────────────────────────┘
```

- **경로 B (경쟁 블로그)**: `(블로그 ID)` + `[현재 블로그 ID 가져오기]`, `(추출 개수=50)`, `[추출 시작]` → 제목 표. **행 클릭 시 곧바로 ②작성 단계로 이동.**
- **경로 C (연관)**: 소스 칩 토글 `◉네이버 ◉구글 ○동영상`, `[추출]` → 연관 키워드·자동완성 제목 표 + `[일괄 내보내기]`.
- 공통: 수집 중 진행률 + `[중단]`(R-1.2). 표 빈/로딩/데이터 상태 필수.
- 선택된 주제는 칩으로 상단 고정 표시 후 `[다음]` 활성화.

### S6 — 삽입·발행 · 컴포넌트 ⑥⑦

```
┌──────────────────────────────┐
│ 발행 옵션                     │
│  ◉ 임시저장(기본)  ○ 바로 발행 │  R-5.1
│ ─ 안내 배너 ─                 │
│ "네이버 글쓰기를 '팝업 모드'로 │  R-4.5
│  열어 주세요."                │
│ [ 네이버 에디터에 삽입 ]      │  주요 버튼
├──────────────────────────────┤
│ 진행: ▓▓▓▓▓▒▒ 70%            │  삽입 작업 순차 진행
│  · 제목 입력 ✓                │
│  · 본문/이미지 삽입 ●          │
│  · 서식 적용 …                │
│ [중단]                        │
└──────────────────────────────┘
```

- 포커스 실패 시 인라인 안내: "본문 영역을 한 번 클릭해 주세요."(R-4.2) + `[재시도]`.
- 편집기 미발견(타임아웃) → 오류 배너 + 안내.
- 완료 → 성공 토스트 "삽입 완료" (임시저장/발행에 따라 문구 분기).

### S7 — 자동 포스팅 · 컴포넌트 ⑧ (모드: 자동)

```
┌──────────────────────────────┐
│ 발행 목록 (3)        [+ 추가]  │
│  1. 캠핑 초보 장비            │  드래그 정렬
│  2. 텐트 고르는 법            │
│  3. 캠핑 요리 …               │
│ 순서  ◉ 목록순  ○ 역순        │  R-6.7
│ 예약  ▢ 사용  [날짜·시각]     │  R-6.5
│ 글 간격  [min]~[max] (무작위) │  R-6.6 계정 보호
├──────────────────────────────┤
│ [ 자동 포스팅 시작 ]          │
├──────────────────────────────┤
│ ─ 진행 패널 ─                 │  R-6.3 실시간
│ 2/3 진행 · 다음 09:40 예정     │
│ ▓▓▓▓▓▓▒▒                     │
│  ✓ 1 발행 완료                │
│  ● 2 삽입 중                  │
│  ○ 3 대기                     │
│ [전체 중단]                   │  R-6.4
└──────────────────────────────┘
```

- 한 글 완료 신호 후에만 다음으로(중복 방지, R-6.2) — UI는 상태만 반영.
- 중단/재개: 세션 영속이라 패널 다시 열어도 진행 상태 유지(R-6.1).

## 7. 마이크로카피 (그대로 사용)

- 주요 버튼: `API 글 생성` · `웹 질문` · `네이버 에디터에 삽입` · `자동 포스팅 시작`
- 보조: `연관 키워드 찾기` · `추출 시작` · `제목+본문 가져오기` · `참조 등록`
- 토스트(성공): `삽입 완료` / `임시저장했어요` / `발행했어요` / `2건 치환했어요`
- 배너(경고): `PC 시계가 정확해야 검색광고 인증이 동작합니다` / `소제목 수와 썸네일 수가 다릅니다`
- 에러: `생성 실패 — 다음 키로 전환해 재시도했어요` / `편집기를 찾지 못했어요. 글쓰기 팝업을 열어 주세요` / `본문 영역을 한 번 클릭해 주세요`
- 빈 상태: `아직 주제가 없어요. 키워드로 시작해 보세요`

## 8. 접근성·반응

- 폭 320~400px 사이에서 **깨지지 않게**(1열 유지, 표는 가로 스크롤 허용).
- 모든 인터랙티브 요소에 포커스 링(키보드 탐색).
- 색만으로 상태를 구분하지 말 것(아이콘/텍스트 병기 — 판정 ⚠ 등).
- 명도 대비는 디자이너가 WCAG AA 이상으로.

## 9. 나노바나나 / 디자이너 인계 노트

- **고정(바꾸지 말 것)**: 화면 목록·각 화면의 영역 구성·컴포넌트 종류·상태 세트·한글 문안·치수 제약(360/720).
- **자유(마음껏)**: 색 팔레트·타이포그래피·아이콘 스타일·일러스트·라운드·그림자·여백 디테일·모션.
- **주의**: ① 좁은 Side Panel(360px)이 기본 캔버스다. 와이드 웹페이지로 그리지 말 것(설정만 720). ② 본문 미리보기에 `[[PF:...]]` 마커를 그리지 말 것(사용자에게 숨김). ③ 진행/상태/에러 화면을 "예쁜 정상 화면"만 그리지 말고 **로딩·빈·에러까지** 세트로.
- 산출물 권장: 컴포넌트 라이브러리(§4) 먼저 → 셸(S0) → 화면 S1~S7 → 상태 변형.

## 10. 열린 항목 (디자인 착수 전 합의)

- IA: 스텝 위저드(제안) vs 상단 탭 — 좁은 폭 기준 위저드 권장.
- 설정을 Options 페이지(제안)로 둘지, 패널 내 탭으로 넣을지.
- 다크 모드 지원 여부.
- 부가요소(④)를 S3에 통합(제안)할지 별도 화면으로 분리할지.
