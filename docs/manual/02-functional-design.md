# 펜플로우 (PenFlow) — 기능 설계서

> **목적**: 네이버 블로그 AI 글쓰기 자동화 도구 **펜플로우**의 동작을 정의한다 — 사용자가 입력한 주제로 AI가 블로그 본문·이미지를 생성하고, SEO 검증을 거쳐 네이버 블로그 편집기에 자동으로 입력·발행한다.
> **독자**: 펜플로우를 구현·유지보수하는 개발자.
> **범위 원칙**: 특정 언어·라이브러리·벤더에 종속되지 않는 **기능/동작 수준**으로 기술한다. (구현 기술은 자유)
> **시나리오 한정**: **네이버 블로그** 단일 시나리오에 집중한다. (타 플랫폼 동시 게시, 영상/GIF 제작 등은 본 문서 범위 밖)

---

## 1. 개요

### 1.1 한 줄 정의
"주제 키워드"를 입력하면 → AI가 SEO 지향 블로그 본문을 생성하고 → 본문을 검증·다듬고 → 썸네일·이미지를 만들고 → 부가 요소(링크·표)를 합성한 뒤 → **네이버 블로그 편집기에 사람처럼 자동 입력하고 → 임시저장 또는 발행**하는 기능.

### 1.2 핵심 가치
- 글감 발굴부터 발행까지의 반복 작업을 자동화한다.
- 단순 텍스트가 아니라 **이미지·소제목 썸네일·표가 결합된 완성형 포스팅**을 만든다.
- **네이버 검색 노출에 유리하도록**(과최적화 방지·중복 회피·광고표기) 보정한다.
- 한 번 설정해두면 **여러 글을 연속/예약 발행**할 수 있다.

### 1.3 설계 전제
- 사용자는 네이버 블로그 계정에 **로그인된 상태**다.
- 본문·이미지 생성은 **외부 생성형 AI 모델**에 위임한다. (구체 벤더는 교체 가능해야 한다)
- 네이버 블로그 편집기는 **외부 시스템**이며 우리가 제어할 수 없다. UI 변경에 대비한 방어 로직이 필요하다.

## 2. 용어 정의

| 용어 | 의미 |
|---|---|
| **주제 키워드** | 글의 소재. 사용자가 직접 입력하거나 주제 선정 단계에서 고른다. |
| **프롬프트** | AI에게 글을 작성시키기 위한 지시문. 저장·재사용 가능. |
| **본문 페이로드(Payload)** | 생성된 본문 HTML + 이미지 + 부가 요소 + 발행 옵션을 묶은 전달 단위. |
| **부가 요소** | 본문에 끼워 넣는 쇼핑 링크·상품표·CTA 버튼·백링크 블록·광고 문구. |
| **비주얼** | 본문에 들어가는 시각 자료 — 대표 이미지, **소제목(H2) 썸네일**, 본문 이미지. |
| **소제목(H2) 썸네일** | 각 소제목 위에 들어가는 제목 텍스트가 얹힌 이미지 카드. 가독성·체류시간용. |
| **마커** | 본문 HTML 안에서 비주얼·부가요소가 들어갈 자리를 나타내는 자리표시 토큰. |
| **삽입 엔진** | 페이로드를 받아 네이버 편집기에 실제로 입력하는 컴포넌트. |
| **형태소 분석** | 본문을 단어 단위로 쪼개 **키워드 반복 횟수/밀도**를 측정하는 검증 기능. |
| **중복 이미지 회피** | 이미지 메타데이터·픽셀을 미세 변형해 네이버의 **유사문서/중복이미지 탐지**를 피하는 처리. |
| **자동 포스팅 세션** | 여러 글을 연속/예약으로 발행하는 작업 단위와 그 상태. |

## 3. 액터(Actor)

| 액터 | 역할 |
|---|---|
| **사용자** | 키, 주제, 프롬프트를 입력하고 생성/발행을 트리거한다. |
| **AI 본문 생성 서비스** | 프롬프트를 받아 본문을 반환하는 외부 시스템. |
| **AI 이미지 생성 서비스** | 프롬프트로 이미지를 생성하는 외부 시스템. |
| **외부 이미지 소스** | 검색형 무료 이미지를 제공하는 외부 시스템. |
| **네이버 블로그 편집기** | 글이 최종적으로 입력·저장·발행되는 외부 대상 시스템. |

## 4. 기능 구성 (논리 컴포넌트)

> 파일·언어가 아닌 **책임 단위**로 구분한다. 하나의 모듈로 합쳐도, 여러 개로 쪼개도 된다.
> ⑨·⑩은 이번 개정에서 추가된 네이버 특화 컴포넌트다.

```
┌──────────────────────────────────────────────────────────────────────┐
│  ① 설정 관리자          키/모델/옵션 입력·저장·검증·내보내기/불러오기        │
├──────────────────────────────────────────────────────────────────────┤
│  ② 주제 선정기          키워드 분석 / 경쟁 블로그 제목 / 연관 키워드 추출    │
├──────────────────────────────────────────────────────────────────────┤
│  ③ 본문 생성기          프롬프트 조립 → AI 호출 → 본문 HTML 수신          │
├──────────────────────────────────────────────────────────────────────┤
│  ⑩ 본문 검증·편집기 ★   형태소 분석(밀도 검증) + 찾기·바꾸기 + HTML 변환    │
├──────────────────────────────────────────────────────────────────────┤
│  ⑨ 비주얼 생성기   ★    대표 이미지·소제목 썸네일·본문 이미지 + 중복 회피    │
├──────────────────────────────────────────────────────────────────────┤
│  ④ 부가요소 합성기       링크·표·CTA·광고문구를 본문 마커 위치에 결합        │
├──────────────────────────────────────────────────────────────────────┤
│  ⑤ 페이로드 저장소       본문/이미지/옵션을 삽입 엔진에 전달                │
├──────────────────────────────────────────────────────────────────────┤
│  ⑥ 삽입 엔진            편집기 탐색 → 자동 입력 → 스타일 적용              │
├──────────────────────────────────────────────────────────────────────┤
│  ⑦ 발행 처리기          임시저장 / 발행 + 완료 신호                       │
├──────────────────────────────────────────────────────────────────────┤
│  ⑧ 자동화 오케스트레이터   연속·예약 세션 상태·진행·다음 글 전이             │
└──────────────────────────────────────────────────────────────────────┘
                                                          ★ = 이번 개정 추가
```

## 5. 메인 시나리오 (정상 흐름)

```
[0]설정    [1]주제선정   [2]본문생성   [3]검증·편집   [4]비주얼생성   [5]부가합성   [6]삽입       [7]발행
 키저장 ─► 주제확정 ─►  AI 본문작성 ─► 밀도검증/수정 ─► 썸네일·이미지 ─► 링크·표·광고 ─► 편집기입력 ─► 저장/발행
(최초1회) (키워드/제목)  (HTML반환)   (형태소·치환)  (생성+중복회피)  (마커결합)    (제목+본문+스타일)
                                                                          └───[8] 자동화: 다음 글 반복/예약───┘
```

- 각 단계 상세는 6~15장.
- **[3]검증·편집**과 **[4]비주얼생성**은 선택적으로 건너뛸 수 있으나, 네이버 노출 품질을 위해 기본 수행을 권장한다.

## 6. [단계 0] 설정 관리 — 컴포넌트 ①

### 6.1 기능
사용자가 기능 동작에 필요한 인증 정보와 옵션을 입력·저장한다.

### 6.2 입력 항목
| 항목 | 필수 | 용도 |
|---|---|---|
| AI 본문 생성 인증 키 | ✅ | 본문 생성 |
| AI 모델 선택 | ✅ | 품질/속도 결정 |
| AI 이미지 생성 인증 키 | 선택 | 대표/소제목/본문 이미지 |
| 외부 이미지 소스 인증 키 | 선택 | 검색형 이미지 |
| 키워드 분석용 인증 정보 | 선택 | 주제 선정(검색량·경쟁도) |
| 제휴/쇼핑 인증 정보 | 선택 | 백링크·상품표 |

### 6.3 비즈니스 규칙
- **R-0.1** AI 인증 키는 **복수 등록**을 허용한다.
- **R-0.2** 사용 중인 키가 **사용량 한도 초과 시 다음 키로 자동 전환**한다. 모든 키 소진 시 명확한 오류를 반환한다.
- **R-0.3** 인증 정보는 **로컬에만 저장**하고 외부로 전송하지 않는다. (생성 요청 시 해당 서비스로의 호출은 예외)
- **R-0.4** 설정 일체를 **내보내기/불러오기**(파일·클립보드·클라우드)할 수 있어야 한다.
- **R-0.5** 시간 기반 서명 인증을 쓰는 외부 API는 **로컬 시계 오차**에 민감하므로, 인증 실패 시 시계 확인을 안내한다.

### 6.4 검증
- 저장 시 형식 검증(빈 값/공백 제거), 저장 성공/실패를 사용자에게 피드백.

## 7. [단계 1] 주제 선정 — 컴포넌트 ②

글감(주제 키워드 또는 글 제목)을 확정한다. **세 경로 중 하나 이상**을 제공한다.

### 7.1 경로 A — 키워드 분석
1. 사용자가 **메인 키워드**를 입력한다.
2. 연관/자동완성 키워드를 수집한다.
3. (인증 정보가 있으면) 각 키워드의 **검색량·경쟁도**를 부가 정보로 제공한다.
4. 사용자가 키워드를 선택 → 주제로 확정.
5. 키워드 목록은 **일괄 입출력**(업로드/다운로드)을 지원한다.

### 7.2 경로 B — 경쟁 블로그 제목 수집
1. 사용자가 대상 블로그 식별자를 입력(또는 현재 페이지에서 자동 인식)한다.
2. 수집할 **게시물 개수**를 지정한다.
3. 해당 블로그의 **게시물 제목 목록**을 수집해 표로 제공한다.
4. **제목을 선택하면 단계 2(본문 생성)로 그 제목이 전달**된다.

### 7.3 경로 C — 연관 키워드 추출 (다중 소스)
1. 사용자가 기준 키워드를 입력한다.
2. **여러 소스**(네이버·구글·동영상 플랫폼 자동완성/연관검색)에서 연관 키워드를 동시에 끌어온다.
3. 어떤 소스를 켤지 **선택**할 수 있다(소스별 on/off).
4. 연관 키워드뿐 아니라 **자동완성 기반 제목 추천**도 함께 제공한다(클릭 후보 발굴).
5. 결과를 표로 제공하고, 선택·일괄 내보내기를 지원한다.
6. 선택된 키워드/제목 → 단계 2로 전달.

### 7.4 비즈니스 규칙
- **R-1.1** 어떤 경로든 결과물은 단계 2의 입력인 **"주제 키워드 또는 제목"** 하나로 수렴한다.
- **R-1.2** 수집 작업은 **중단(취소) 가능**해야 하며, 진행률을 표시한다.
- **R-1.3** 경로 C의 소스는 **추가/교체 가능**하도록 소스 어댑터로 추상화한다.

## 8. [단계 2] 본문 생성 — 컴포넌트 ③

### 8.1 입력
- 주제 키워드(또는 제목)
- 프롬프트 (저장된 것 선택 또는 직접 작성)
- (선택) 참고 자료 — 현재 보고 있는 페이지 본문 크롤링 결과("제목+본문 가져오기")

### 8.2 처리
1. **프롬프트 조립**: 주제 + 사용자 프롬프트 + (선택)참고자료를 하나의 지시문으로 합친다.
2. **AI 호출**: 인증 키와 선택 모델로 생성 요청.
3. **응답 후처리**: 코드블록 마크(```)·설명 텍스트 등 **불필요한 래퍼 제거**.
4. **마커 삽입 보장**: 본문에는 비주얼·부가요소가 들어갈 자리를 나타내는 **자리표시 마커**가 포함되어야 한다. (예: 소제목, 이미지 자리, 상품표 자리, 백링크 자리, 광고문구 자리)

### 8.3 생성 방식 (택1, 둘 다 제공 권장)
| 방식 | 특징 | 인증 키 |
|---|---|---|
| **직접 호출 방식** | 빠름, 안정적 | 필요 |
| **대화형 서비스 자동화 방식** | 키 불필요, 느림(대기시간 설정) | 불필요 |

### 8.4 비즈니스 규칙
- **R-2.1** 프롬프트는 **이름을 붙여 저장/불러오기/삭제**할 수 있다.
- **R-2.2** 본문은 **구조화된 형식**(소제목 단위 구획)을 따라야 비주얼·부가요소 합성이 가능하다.
- **R-2.3** 생성 실패(키 소진·네트워크·빈 응답) 시 재시도 또는 다음 키 전환 후, 그래도 실패하면 사용자에게 사유를 명시한다.
- **R-2.4** 출력 형식 규약(마커 종류·소제목 개수 등)은 프롬프트와 하위 단계 간 **계약(contract)** 으로 고정한다.
- **R-2.5** 키워드별/제목별로 **클릭 시 자동 생성**(일괄 트리거) 옵션을 제공할 수 있다.

### 8.5 출력
- 마커가 포함된 **본문 HTML**.

## 9. [단계 3] 본문 검증·편집 — 컴포넌트 ⑩ ★신규

> 생성된 본문을 **네이버 노출에 맞게 점검·교정**하고, 삽입 직전 형태를 확정하는 단계.

### 9.1 형태소 분석 (키워드 밀도 검증)
- **목적**: 같은 키워드를 과도하게 반복하면 네이버가 **과최적화(스팸)** 로 판단해 노출이 떨어진다. 이를 사전에 잡는다.
- **동작**:
  1. 본문을 형태소(단어) 단위로 분해한다.
  2. 주요 키워드의 **출현 횟수·비율(밀도)** 을 집계해 표로 보여준다.
  3. 권장 범위를 벗어난 키워드를 **경고 표시**한다.
- **결과 활용**: 사용자는 경고를 보고 아래 "찾기·바꾸기"로 조정한다.

```
예시 출력
┌──────────────┬───────┬────────┬──────────┐
│ 키워드        │ 횟수  │ 밀도   │ 판정      │
├──────────────┼───────┼────────┼──────────┤

## 10. [단계 4] 비주얼 생성 — 컴포넌트 ⑨ ★신규

> 본문에 들어갈 **이미지 일체**를 만든다. 네이버 블로그는 이미지·소제목 썸네일이 체류시간·노출에 크게 작용하므로 별도 컴포넌트로 둔다.

### 10.1 비주얼 종류
| 종류 | 설명 | 들어가는 위치 |
|---|---|---|
| **대표(H1) 썸네일** | 글 전체를 대표하는 표지 이미지 | 본문 최상단 |
| **소제목(H2) 썸네일** | 소제목 텍스트가 얹힌 이미지 카드 | 각 소제목 위 |
| **본문 이미지** | 문단 사이 삽화 | 이미지 마커 위치 |

### 10.2 이미지 확보 방식 (모드)
하나의 이미지 슬롯을 아래 방식 중 하나로 채운다. (교체 가능)
| 모드 | 설명 |
|---|---|
| **AI 생성** | 프롬프트로 이미지를 생성한다. |
| **외부 이미지 소스** | 검색형 무료 이미지를 가져온다. |
| **사용자 업로드** | 사용자가 직접 올린 이미지(드래그앤드롭) 사용. |
| **기본/클립보드** | 미리 지정한 기본 이미지 또는 클립보드 이미지 사용. |

### 10.3 소제목(H2) 썸네일 생성
1. 소제목 텍스트를 가져온다.
2. **배경**을 정한다: 업로드 이미지 / 지정 URL / **동영상 한 프레임 캡처** 중 선택.
3. 배경 위에 소제목 텍스트를 합성한다(테두리·폰트 등 옵션).
4. 자동 생성 on/off, 생성 간 **지연시간(min~max)** 설정으로 외부 서비스 부하를 조절한다.

### 10.4 중복 이미지 회피
- **목적**: 같은/유사 이미지를 반복 사용하면 네이버가 **유사문서·중복이미지**로 판단해 노출이 불리하다. 이를 피한다.
- **동작**: 이미지의 **메타데이터를 제거/변형**하고, 필요 시 픽셀을 미세 변형해 매번 **고유한 파일**로 만든다.
- on/off 옵션으로 제공한다.

### 10.5 캐릭터/모델 이미지 일관성
- **목적**: 한 포스팅(또는 여러 포스팅) 안의 AI 생성 이미지에 **같은 인물·캐릭터가 일관되게 등장**하도록 한다. 매번 다른 사람이 나오면 브랜드 정체성이 흐려진다.
- **동작**:
  1. 사용자가 **기준 인물/캐릭터 이미지**를 등록한다(클립보드 붙여넣기 또는 업로드, 초기화 가능).
  2. AI 이미지 생성 요청 시 이 **참조 이미지를 함께 전송**해 동일 인물·화풍을 유지시킨다.
  3. (예: "첫 번째 모델 이미지에 두 번째 상품 이미지를 자연스럽게 합성" 같은 결합 지시 지원)
- 등록된 모델 이미지는 **재사용**되며, 사용자가 해제(초기화)할 때까지 유지된다.

### 10.6 이미지 후처리
- **품질(압축률) 조절**: 슬라이더로 용량/화질 균형을 맞춘다(로컬 저장 용량 관리와 연동).
- 본문 이미지 미리보기·일괄 삭제·다운로드 지원.

### 10.7 비즈니스 규칙
- **R-7.1** 비주얼 생성은 **선택 단계**다. 없으면 텍스트만으로도 발행 가능해야 한다.
- **R-7.2** 이미지 확보 모드는 **슬롯별로 교체 가능**하다(특정 벤더 종속 금지).
- **R-7.3** 소제목 썸네일은 **소제목 개수만큼** 생성하며, 소제목과 1:1로 매칭해 마커에 배치한다.
- **R-7.4** 중복 회피가 켜진 경우, 모든 출력 이미지는 **서로 다른 바이트**를 갖도록 보장한다.
- **R-7.5** 외부 서비스 호출은 **지연시간 설정**으로 속도 제한(rate limit)을 회피한다.
- **R-7.6** 생성된 이미지는 페이로드의 이미지 목록으로 전달되며, 본문 마커와 **순서·개수가 일치**해야 한다.
- **R-7.7** 캐릭터/모델 이미지가 등록돼 있으면 **모든 AI 이미지 생성에 자동 동반**하며, 중복 회피(R-7.4)와 충돌하지 않도록 회피 처리는 동반 후 결과물에 적용한다.

## 11. [단계 5] 부가 요소 합성 — 컴포넌트 ④

### 11.1 기능
본문 HTML의 **마커 위치를 찾아** 순서대로 비주얼·부가 요소를 결합해 **삽입 작업 큐**를 만든다.

### 11.2 부가 요소 종류
| 요소 | 설명 | 비고 |
|---|---|---|
| 이미지/썸네일 | ⑨에서 생성된 비주얼 | 마커와 1:1 |
| 쇼핑/제휴 링크 | 지정한 소제목 위치마다 삽입 | 위치를 사용자가 지정 |
| 상품표 | 상품 정보 표 | **최신 상품표 자동 반영** 옵션 |
| CTA 버튼 | 행동 유도 버튼 | |
| 백링크 블록 | 외부 링크 묶음 | |
| **광고/협찬 문구** | "대가성 표기" 등 정책 준수 문구 | **네이버 정책 준수용** |
| 출처 링크 | 참고 페이지 URL | 옵션 |

### 11.3 처리 로직 (의사코드)
```
큐 = []
본문을 마커 기준으로 순회:
    마커 이전의 일반 텍스트 → 큐에 text 작업 추가
    마커 종류에 따라:
        광고문구  → (옵션 ON이면) 본문 상/하단 지정 위치에 ad-notice 작업 추가
        소제목    → (해당 위치가 쇼핑링크 지정 위치면 link 작업 먼저) text(소제목) 추가
        이미지    → image 작업 추가
        상품표    → table 작업 추가
        백링크    → backlink 작업 추가
마커 인덱스 순서대로 정렬해 최종 큐 확정
```

### 11.4 비즈니스 규칙
- **R-3.1** 마커에 대응하는 리소스가 없으면 **그 마커는 건너뛴다**(빈자리 방지).
- **R-3.2** 요소의 **본문 내 순서**는 마커 위치를 따른다(임의 재배치 금지).
- **R-3.3** 합성 결과는 **순서가 보장된 작업 큐**다. 삽입 엔진은 이 큐를 그대로 실행한다.
- **R-3.4** **광고/협찬 문구**는 옵션이 켜지면 **반드시 누락 없이** 포함한다(정책 준수가 목적이므로 스킵 금지).
- **R-3.5** 상품표 "자동 반영"이 켜진 경우, 합성 시점의 **최신 상품표 데이터**를 사용한다.

## 12. [단계 6] 편집기 자동 삽입 — 컴포넌트 ⑥

### 12.1 기능
페이로드(작업 큐)를 받아 **네이버 블로그 편집기에 사람처럼 자동 입력**한다.

### 12.2 처리 순서
1. **편집기 준비 대기**: 편집 영역이 로드될 때까지 **재시도하며 폴링**(타임아웃 존재).
2. **초기 팝업 정리**: '이어쓰기' 등 방해 팝업을 자동으로 닫는다.
3. **작업 큐 순차 실행**:
   - 텍스트/HTML → 붙여넣기 방식으로 입력
   - 이미지/썸네일 → 입력 후 렌더링 대기, 필요 시 링크 부착
   - 쇼핑 링크 → 입력 후 자동 생성된 불필요 텍스트 제거
   - 표/백링크 블록 → 안정화 대기 후 입력
   - **각 작업 사이에 대기 시간**을 둬 편집기 처리 속도를 맞춘다.
4. **제목 입력**: 본문 첫 소제목 등에서 제목을 추출해 제목란에 입력.
5. **스타일 일괄 적용**: 본문 전체 선택 후 줄간격·글꼴·글자크기 등 기본 서식 적용.

### 12.3 비즈니스 규칙
- **R-4.1** 외부 편집기 UI 의존 지점(요소 탐색)은 **실패에 대비한 폴링·타임아웃·폴백**을 둔다.
- **R-4.2** 편집 영역 포커스가 없으면 **사용자에게 본문 클릭을 안내**하고 일정 시간 재시도한다.
- **R-4.3** 입력 속도는 외부 편집기 처리 지연에 맞춰 **조절 가능**해야 한다(하드코딩 지양 권장).
- **R-4.4** 삽입 중 오류 시, **어느 작업에서 실패했는지** 사용자에게 표시하고 안전하게 종료한다.
- **R-4.5** 편집기 모드(팝업/사이드바)에 따라 동작이 달라질 수 있으므로 **권장 모드를 안내**한다.

## 14. [확장] 연속·예약 자동 포스팅 — 컴포넌트 ⑧

### 14.1 기능
여러 주제/제목에 대해 단계 2~7을 **반복**한다. 예약 시각 발행, **글 간 간격 제어**, 발행 순서 지정을 지원한다.

### 14.2 상태 머신
```
        ┌────────────┐  시작/예약시각 도달   ┌───────────┐
        │   IDLE     │ ───────────────────► │  RUNNING   │
        └────────────┘                      └─────┬─────┘
              ▲                                    │ 한 글: 생성→검증→비주얼→합성→삽입→발행
              │ 모든 글 완료 / 사용자 중단            │
              │                                    ▼
              │                              ┌───────────┐  완료 신호
              └───────────────────────────── │ STEP_DONE │ ──┐
                                             └───────────┘   │ 다음 글 있음
                                                   ▲         │
                                                   └─────────┘
```

### 14.3 비즈니스 규칙
- **R-6.1** 세션 상태(현재 글 인덱스·전체 목록·진행률·옵션)는 **영속 저장**해 중단·재개에 견딘다.
- **R-6.2** 한 글의 **완료 신호를 받은 뒤에만** 다음 글로 전이한다(중복 발행 방지).
- **R-6.3** 진행 상황을 사용자에게 **실시간 표시**한다(진행 패널/상태 메시지).
- **R-6.4** 사용자는 언제든 **중단**할 수 있고, 중단 시 진행 중 작업을 안전하게 종료한다.
- **R-6.5** 예약 발행은 **지정 시각 트리거**로 시작한다.
- **R-6.6 (계정 보호)** 연속 발행 시 글과 글 사이에 **무작위 간격(min~max)** 을 둬 어뷰징/스팸 탐지를 회피한다. 간격은 **설정 가능**해야 하며, 위험 신호(예: 보호 모드 활성)가 감지되면 **간격을 자동으로 더 늘리고**, 해제되면 기본값으로 복구한다.
- **R-6.7** 발행 **순서**(목록 순/역순 등)를 지정할 수 있다.

```
Settings {

    imageSourceCredential?: Credential   // 검색형 이미지
    keywordToolCredential?: Credential
    affiliateCredential?: Credential
}

Prompt { name: string, body: string }

Topic { keyword: string, title?: string, metrics?: {volume, competition} }

// 단계 3 산출
DensityReport { items: [{ keyword, count, density, verdict }] }

// 단계 4 산출
Visual {
    role: H1_THUMB | H2_THUMB | BODY_IMAGE
    source: AI | EXTERNAL | UPLOAD | DEFAULT | CLIPBOARD
    data: BinaryOrRef
    dedupApplied: bool                   // 중복 회피 적용 여부
    h2Caption?: string                   // 소제목 썸네일의 텍스트

Payload {

    publishOption: TEMP_SAVE | PUBLISH
}

InsertTask { type: text|image|shoppingLink|productTable|backlink|ctaButton|adNotice, content }
InsertQueue = [InsertTask]               // 순서 보장

AutoPostSession {
    state: IDLE | RUNNING | STEP_DONE
    items: [Topic]
    currentIndex: int
    order: LIST | REVERSE | ...          // 발행 순서
    scheduleTime?: datetime
    postIntervalMin: duration            // 글 간 최소 간격 (계정 보호)
    postIntervalMax: duration            // 글 간 최대 간격 (무작위 상한)

## 16. 인터페이스 계약 (컴포넌트 간)

| 출발 | 도착 | 전달물 | 계약 |
|---|---|---|---|
| ② 주제 선정 | ③ 본문 생성 | Topic | 키워드 또는 제목 1건 |
| ③ 본문 생성 | ⑩ 검증·편집 | 본문 HTML | **약속된 마커** 포함 |
| ⑩ 검증·편집 | ⑨ 비주얼 생성 | 본문 HTML(보정본) | 마커 보존 |
| ⑨ 비주얼 생성 | ④ 합성 | Visual 목록 | 마커와 **순서·개수 일치** |
| ④ 합성 | ⑥ 삽입 엔진 | InsertQueue | 순서 보장 큐 |
| ⑥ 삽입 엔진 | ⑦ 발행 | 발행 옵션 | 삽입 완료 후 호출 |
| ⑦ 발행 | ⑧ 오케스트레이터 | 완료 신호 | 글 1건 종료 통지 |

> **핵심 계약 1 — 마커 규약**: ③이 심는 마커의 종류·표기를 ⑨·④가 그대로 해석한다. 마커가 바뀌면 세 컴포넌트를 함께 수정해야 한다.
> **핵심 계약 2 — 비주얼 정합성**: ⑨의 Visual 개수·순서가 본문 이미지 마커와 어긋나면 이미지가 엉뚱한 자리에 들어간다. 합성 전 **정합성 검사**를 둘 것.

## 17. 예외/엣지 케이스

| 상황 | 처리 |
|---|---|
| AI 키 한도 초과 | 다음 키로 전환, 전부 소진 시 사용자에 통지 |
| AI 빈 응답/형식 오류 | 후처리로 복구 시도 → 실패 시 재생성 또는 중단 |
| 형태소 밀도 과다 | 경고 표시, 찾기·바꾸기 유도(자동 차단은 아님) |
| 이미지 생성 실패 | 해당 슬롯을 대체 모드로 폴백하거나 비워 둠(본문은 진행) |
| 소제목 수 ≠ 썸네일 수 | 합성 전 정합성 검사로 차단, 사용자에 불일치 통지 |
| 중복 회피 적용 후 동일 바이트 잔존 | 재변형 1회 더 시도, 실패 시 경고 |
| 편집기 영역 미발견 | 폴링·타임아웃 후 사용자에 포커스/모드 안내 |
| 마커 대응 리소스 없음 | 해당 마커 스킵 (단, 광고문구는 스킵 금지) |
| 외부 편집기 UI 변경 | 폴백 셀렉터·명확한 오류 메시지 |
| 삽입 중 사용자 중단 | 진행 작업 정리 후 안전 종료 |
| 자동 세션 중복 발행 | 완료 신호 수신 후에만 다음 글 진행 |
| 로컬 시계 오차 | 서명 인증 실패 시 시간 확인 안내 |
| 로컬 저장 용량 부족 | 저장 용량 미터로 사전 경고, 이미지 품질↓ 유도 |

## 18. 비기능 요구사항

- **이식성**: AI 벤더·이미지 소스·저장 방식을 **교체 가능**하도록 인터페이스로 추상화한다.
- **복원력**: 외부 편집기/네트워크 불안정에 대해 재시도·타임아웃·폴백을 기본 탑재.
- **관측성**: 단계별 진행/실패를 사용자와 로그 양쪽에 남긴다(진행 패널 포함).
- **보안/프라이버시**: 인증 정보는 로컬 보관, 평문 노출·하드코딩 금지.
- **저장 용량 관리**: 생성 이미지가 로컬 저장을 점유하므로 **사용량 미터**를 제공하고, **이미지 품질(압축)** 로 용량을 조절한다.
- **정책 준수**: 광고/협찬 표기를 옵션으로 보장해 네이버 정책 위반을 방지한다.
- **계정 보호**: 연속 발행 시 무작위 포스팅 간격을 둬 어뷰징 탐지를 회피하고, 위험 감지 시 간격을 자동 확대한다.
