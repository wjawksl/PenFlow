# 펜플로우 (PenFlow) — 사용 시나리오

> **펜플로우**로 네이버 블로그 글을 **주제 발굴부터 발행까지** 어떻게 완성하는지, 사용자 관점의 전체 흐름을 정리한 문서.
> ※ 이 문서는 펜플로우가 **목표로 하는 사용 흐름**을 담는다 — 특정 화면·버튼이 아니라 *사용자가 무엇을 하는지* 기준이다.
> 기능·규칙·데이터 설계는 `02-functional-design.md`, 구현 기술은 `04-tech-stack.md`를 참조한다.

펜플로우는 네이버 블로그 글쓰기를 자동화하는 **Chromium 브라우저 확장**이다. 주제를 정하면 AI가 본문과 이미지를 만들고, 검색 노출에 맞게 다듬은 뒤, 네이버 에디터에 자동으로 입력·발행한다. (지원 브라우저: 크롬·네이버 웨일·엣지 등 — 근거는 `04-tech-stack` 참조)

---

## 전체 흐름 한눈에 보기

```
[0] 키 설정 ──► [1] 주제 선정 ──► [2] 글 생성 ──► [3] 자동 삽입 ──► [4] 발행
  (최초 1회)    키워드·제목 확보    본문+이미지+부가요소    네이버 에디터       임시저장 / 발행
                                  (검증·다듬기 포함)        사람처럼 입력
                                                                  └──[확장] 다음 글 반복 / 예약──┘
```

| 단계 | 하는 일 | 핵심 산출 |
|---|---|---|
| 0. 키 설정 | 인증 키·옵션 입력·저장 (최초 1회) | 사용 준비 완료 |
| 1. 주제 선정 | 키워드·제목 확보 | 글의 주제 1건 |
| 2. 글 생성 | AI 본문 + 이미지 + 부가요소 결합 | 발행 직전 완성본 |
| 3. 자동 삽입 | 네이버 에디터에 자동 입력 | 에디터에 올라간 글 |
| 4. 발행 | 임시저장 또는 발행 | 게시 완료 |

> 2단계 "글 생성"에는 본문 작성뿐 아니라 **키워드 밀도 검증·다듬기, 썸네일·이미지 생성, 링크·표·광고문구 결합**이 함께 일어난다. 세부 컴포넌트(①~⑩) 구분은 `02-functional-design`를 참조한다.

---

## 0단계 — 키 설정 (최초 1회)

설정 화면에서 펜플로우 동작에 필요한 인증 키와 옵션을 입력·저장한다.

**반드시 필요한 것**
- **AI 본문 생성용 인증 키** (예: 구글 Gemini 등) — 본문을 만드는 데 쓰인다.
- **AI 모델 선택** — 품질과 속도를 정한다. (상위 모델일수록 품질↑·속도↓)

**선택 (주제 선정·부가 기능용)**
| 항목 | 용도 |
|---|---|
| 키워드 분석용 인증 정보 | 검색량·경쟁도 측정 |
| 검색 결과 수집용 인증 정보 | 블로그·뉴스 검색 결과 가져오기 |
| AI/외부 이미지 인증 정보 | 본문·썸네일 이미지 확보 |
| 제휴·쇼핑 인증 정보 | 상품표·백링크 |

알아둘 점:
- **AI 키는 여러 개 등록 가능**하며, 할당량이 초과되면 자동으로 다음 키로 전환된다.
- 모든 인증 정보는 **사용자 PC(로컬)에만 저장**되고, 펜플로우가 외부 서버로 보내지 않는다(해당 AI·검색 서비스로의 호출은 예외).
- 설정 일체를 **내보내기/불러오기**(파일·클립보드·클라우드)할 수 있다.
- 시간 기반 서명 인증을 쓰는 일부 키는 **PC 시계가 정확해야** 동작하므로, 인증 실패 시 시간을 확인한다.

---

## 1단계 — 주제 선정

글감(주제 키워드 또는 글 제목)을 확정한다. 세 가지 경로 중 하나 이상을 쓸 수 있다.

**경로 A — 키워드에서 출발**
메인 키워드를 넣으면 연관·자동완성 키워드를 모으고, (인증 정보가 있으면) 키워드별 검색량·경쟁도를 표로 보여준다. 좋은 키워드를 골라 주제로 삼는다. 키워드 목록은 일괄 업로드/다운로드할 수 있다.

**경로 B — 경쟁 블로그에서 출발**
대상 블로그를 지정하고 수집할 개수를 정하면, 그 블로그의 게시물 제목 목록을 모아 표로 보여준다. **제목을 고르면 그대로 2단계(글 생성)로 넘어간다.**

**경로 C — 연관 키워드 추출**
기준 키워드로 여러 소스(네이버·구글·동영상 등)에서 연관 키워드와 자동완성 기반 제목을 동시에 끌어온다. 소스별 on/off가 가능하고, 결과를 표로 정리·일괄 내보낼 수 있다.

> 어느 경로를 쓰든 결과는 2단계의 입력인 **"주제 키워드 또는 제목" 하나**로 모인다. 수집 작업은 언제든 중단할 수 있고 진행률이 표시된다.

---

## 2단계 — 글 생성

**입력**
- 1단계에서 고른(또는 직접 입력한) **주제 키워드/제목**
- **프롬프트** — AI에게 줄 지시문. 이름을 붙여 저장·재사용·삭제할 수 있다.
- (선택) **참고 자료** — 현재 보고 있는 페이지 본문을 가져와 참고로 쓸 수 있다.

**생성 방식 (둘 중 택1)**
- **직접 호출** — 인증 키로 AI를 바로 호출한다. 빠르고 안정적이며, 키 할당량이 차면 다음 키로 자동 전환된다.
- **대화형 자동화** — AI 서비스 화면을 자동 조작해 생성한다. 키가 없어도 되지만 느리다(대기시간 설정).

**자동으로 함께 처리되는 것 (선택 가능)**
- **검증·다듬기** — 키워드 반복이 과한지 밀도를 점검하고, 찾기·바꾸기로 조정한다. (과최적화로 인한 노출 불이익 방지)
- **비주얼** — 대표 썸네일, 소제목별 썸네일, 본문 이미지를 만든다. 같은 이미지 반복으로 인한 불이익을 피하고, 인물·캐릭터를 일관되게 유지할 수 있다.
- **부가요소** — 쇼핑/제휴 링크, 상품표, CTA 버튼, 백링크, **광고·협찬 문구**를 정해진 위치에 결합한다.

결과는 발행 직전 완성본(본문 + 이미지 + 부가요소 + 발행 옵션)으로 묶여 다음 단계로 넘어간다.

---

## 3단계 — 네이버 에디터 자동 삽입

펜플로우가 네이버 글쓰기 페이지에서 에디터를 **사람처럼 자동 조작**한다. 본문 텍스트·이미지·표·링크를 정해진 순서대로 넣고, **제목을 입력**하며, 줄간격·글꼴·글자크기 같은 **서식을 일괄 적용**한다.

> 외부 에디터라 화면 상태에 따라 동작이 달라질 수 있어, 펜플로우가 **권장 편집기 모드**를 안내한다. 입력이 시작되지 않으면 본문 영역을 한 번 클릭해 포커스를 주면 된다.

---

## 4단계 — 발행

선택한 옵션에 따라 마무리한다. (기본값은 안전하게 **임시저장**)
- **임시저장** — 임시 보관 후 "삽입 완료"를 알린다.
- **발행** — 발행과 확인까지 자동으로 처리한다.

---

## (확장) 연속·예약 자동 포스팅

여러 주제에 대해 2~4단계를 반복한다.
- 예약한 시각에 발행할 수 있다.
- 글과 글 사이에 **무작위 간격**을 둬 계정을 보호하며, 위험이 감지되면 간격을 자동으로 더 늘린다.
- 발행 순서를 지정할 수 있고, 한 글이 끝났다는 신호를 받은 뒤에야 다음 글로 넘어가 **중복 발행을 막는다.**
- 진행 상황은 패널로 실시간 표시된다.

## 부록 — 핵심 입력 요약

| 구분 | 최소 필요 |
|---|---|
| 글 생성 (필수) | AI 본문 생성 키 1개 + 모델 선택 |
| 주제 선정 (선택) | 키워드 분석·검색용 인증 정보 |
| 부가 기능 (선택) | 이미지·제휴 인증 정보 |

> 펜플로우의 심장은 **에디터 자동 삽입**(3단계)이다. 실제로 네이버 에디터에 글·이미지·발행까지 처리하는 부분이라, 기능 개선의 중심이 된다. 동작 규칙과 예외 처리는 `02-functional-design`, 구현 기술 선택은 `04-tech-stack`을 참조한다.
