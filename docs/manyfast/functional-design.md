## 1. 프로젝트 개요
---
### 한 줄 정의
네이버 블로그 자동 포스팅 확장

### 제품 목표
네이버 블로그 글쓰기를 주제 발굴부터 발행까지 자동화해, 반복 작업 시간을 줄이고 SEO 품질과 정책 준수를 안정적으로 달성한다.

### 배경
네이버 블로그 글쓰기는 주제 발굴→본문 작성→이미지/썸네일 제작→링크/표/광고문구 삽입→에디터 입력→임시저장/발행까지 반복 작업이 많다. 외부 AI로 텍스트/이미지를 만들더라도 네이버 SmartEditor에 옮겨 넣는 과정과 SEO 검증, 정책 준수(광고/협찬 표기), 연속/예약 발행의 계정 보호(간격 제어) 같은 운영 부담이 남는다. 또한 인증 키는 로컬 보관이 요구되며, 서버 없이 클라이언트만으로 동작해야 한다.

### 사용자 문제
- 주제 발굴, 본문 생성, 이미지 제작, 부가요소(제휴/상품표/CTA/백링크/광고문구) 결합, 네이버 에디터 입력/발행까지 과정이 길고 반복적이다.
- 과도한 키워드 반복(과최적화), 이미지 중복, 광고/협찬 문구 누락 등으로 노출/정책 리스크가 발생한다.
- 네이버 편집기는 외부 시스템이라 UI 변화/지연으로 자동화가 깨지기 쉽고, 연속 발행 시 계정 보호가 필요하다.
- AI/검색/이미지 등 다양한 인증 키를 안전하게 관리하고, 한도 초과 시 자동 전환이 필요하다.

### 해결 방안
Chromium 브라우저 확장(Manifest V3) 형태로 동작하는 ‘펜플로우’를 제공한다.
- ① 설정: AI/검색/이미지/제휴 인증 키를 로컬에 저장, 복수 키 등록 및 한도 초과 시 자동 순환, 내보내기/불러오기 지원
- ② 주제 선정: 키워드 분석/경쟁 블로그 제목 수집/다중 소스 연관 키워드 추출로 주제(키워드 또는 제목) 1건을 확정
- ③ 본문 생성: 저장된 프롬프트와 참고자료(현재 페이지)로 AI 본문을 생성하고, 마커 규약에 맞게 후처리
- ⑩ 검증·편집: 형태소 기반 키워드 밀도 점검 및 찾기·바꾸기, HTML↔마크다운 변환/미리보기로 최종 본문 확정
- ⑨ 비주얼 생성: 대표/H2 썸네일/본문 이미지 생성(외부/업로드/기본 포함), 중복 이미지 회피, 모델 일관성 유지
- ④ 합성: 마커 위치에 링크/표/CTA/백링크/광고문구를 결합해 삽입 작업 큐 생성
- ⑥⑦ 삽입·발행: 네이버 SmartEditor를 사람처럼 자동 조작해 입력 후 임시저장/발행
- ⑧ 자동화: 여러 주제에 대해 연속/예약 발행, 무작위 간격 및 위험 감지 시 간격 자동 확대, 진행 패널 제공

### 차별점
- ‘에디터 자동 삽입(삽입 엔진)’을 중심으로 네이버 SmartEditor에 실제 입력·서식·발행까지 자동화(단순 글 생성기와 차별)
- 마커 규약 기반으로 본문/비주얼/부가요소를 정합성 검사하며 결합(순서·개수 불일치 차단)
- 형태소 분석(로컬)로 키워드 밀도 검증 및 편집 지원(과최적화 리스크 완화)
- 이미지 중복 회피(바이트 고유성 보장) 및 모델 일관성(참조 이미지) 지원
- 인증 키 로컬 전용 저장 및 복수 키 자동 순환, 서버 없는 구조로 프라이버시 강화
- 연속/예약 발행 시 무작위 간격 및 위험 감지 기반 계정 보호 로직 내장

### 타겟 사용자
- 네이버 블로그를 운영하며 글을 반복 생산하는 개인/소규모 팀(마케터, 1인 사업자, 제휴/리뷰 블로거)
- 크롬/웨일/엣지 등 Chromium 브라우저로 블로그를 작성하는 사용자
- 생성형 AI를 활용하되, 키 관리·SEO 과최적화·이미지 중복·광고 표기 등 운영 리스크를 줄이고 싶은 사용자

### 사용자 시나리오
1) 최초 1회: 사용자가 Options에서 AI 본문 키/모델을 설정하고(필수), 필요 시 이미지/키워드분석/제휴 키를 추가한 뒤 저장한다. 키는 여러 개 등록하고, 내보내기/불러오기로 설정을 이동한다.
2) 사용자는 Side Panel에서 주제 선정 단계로 이동해 (A)키워드 분석, (B)경쟁 블로그 제목 수집, (C)다중 소스 연관 키워드 추출 중 하나로 주제(키워드 또는 제목) 1건을 확정한다.
3) 작성 단계에서 프롬프트를 선택/편집하고(저장/재사용), 필요 시 현재 페이지에서 참고자료를 가져온 뒤 본문 생성을 실행한다(직접 호출 또는 대화형 자동화).
4) 선택적으로 검증·편집 단계에서 형태소 분석으로 키워드 밀도를 확인하고, 찾기·바꾸기 및 HTML↔마크다운 변환/미리보기로 최종 본문을 확정한다.
5) 선택적으로 비주얼 단계에서 대표/H2 썸네일/본문 이미지를 생성(또는 업로드)하고, 중복 이미지 회피/모델 일관성을 적용한다.
6) 삽입·발행 단계에서 네이버 글쓰기 팝업을 열고 ‘삽입’을 누르면, 펜플로우가 본문/이미지/표/링크를 순서대로 입력하고 서식을 일괄 적용한 뒤 임시저장 또는 발행한다.
7) 확장: 자동 포스팅 모드에서 여러 주제를 목록으로 관리하고 예약 시각/발행 순서/무작위 간격을 설정해 연속 발행한다. 진행 패널에서 상태를 확인하고 중단/재개한다.

### 핵심 지표
- 글 1건 완성 소요시간(주제 확정→임시저장/발행) 절감률

- 사용자 재사용 지표: 저장 프롬프트 사용률, 연속/예약 기능 사용률

### 리스크

## 1. 설정 및 인증 키 관리
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전

AI/검색/이미지/제휴 인증 정보를 로컬에 안전하게 저장하고, 복수 키 등록·자동 순환 및 내보내기/불러오기를 제공한다.

**수용 기준:**
```
1. ○ AI 본문 생성 키를 여러 개 등록/삭제할 수 있다.
2. ○ 사용량 한도 초과 시 다음 키로 자동 전환되며, 모든 키 소진 시 명확한 오류를 표시한다.
3. ○ 설정(키/옵션)을 파일 또는 클립보드로 내보내고 불러올 수 있다.
4. ○ 인증 정보는 로컬에만 저장되며, 펜플로우 서버로 전송하지 않는다.
```


### 1.1 로컬 설정 저장 및 기본 검증
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: 사용자 | **기기**: Chromium 브라우저 확장(크롬/네이버 웨일/엣지)

Options에서 인증 키/모델/옵션을 입력·저장하고, 빈 값/형식 오류를 검증해 사용자에게 피드백한다.


#### 1.1.1 설정 항목 입력/저장/피드백

AI 본문 키/모델(필수) 및 선택 키를 입력하고 저장 성공/실패 토스트를 제공한다. 빈 값/공백 제거 등 기본 검증을 수행한다.
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: 사용자 | **기기**: Chromium 브라우저 확장(크롬/네이버 웨일/엣지)


### 1.2 AI 키 복수 등록 및 자동 순환
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: 사용자 | **기기**: Chromium 브라우저 확장(크롬/네이버 웨일/엣지)

AI 본문 생성 키를 복수로 관리하고, 호출 실패/한도 초과 시 다음 키로 자동 전환한다.


#### 1.2.1 키 소진 감지 및 자동 전환 로직

AI 호출 시 한도 초과/인증 실패를 감지하면 다음 키로 전환해 재시도한다. 모든 키 실패 시 사용자에게 사유를 표시한다.
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: 사용자 | **기기**: Chromium 브라우저 확장(크롬/네이버 웨일/엣지)


### 1.3 설정 내보내기/불러오기
> **중요도**: 🟡 중간 | **진행 상태**: ⚪ 시작전 | **역할**: 사용자 | **기기**: Chromium 브라우저 확장(크롬/네이버 웨일/엣지)

설정 전체를 파일 또는 클립보드로 내보내고, 다시 불러와 복원할 수 있다.


#### 1.3.1 설정 내보내기/불러오기 포맷

설정을 파일/클립보드로 직렬화해 내보내고, 불러오기 시 덮어쓰기 여부를 확인한 뒤 복원한다.
> **중요도**: 🟡 중간 | **진행 상태**: ⚪ 시작전 | **역할**: 사용자 | **기기**: Chromium 브라우저 확장(크롬/네이버 웨일/엣지)

## 2. 주제 선정(키워드/경쟁 블로그/연관 추출)
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전

주제 키워드 또는 제목 1건을 확정하기 위해 3가지 경로(A/B/C)의 수집·표시·선택 흐름을 제공한다.

**수용 기준:**
```
1. ○ 키워드 분석 경로에서 연관/자동완성 키워드를 수집하고 목록을 표로 표시한다.
2. ○ 경쟁 블로그 경로에서 게시물 제목 목록을 수집해 표로 표시하고, 제목 선택 시 다음 단계 입력으로 전달한다.
3. ○ 다중 소스 연관 추출 경로에서 소스별 on/off로 수집하고 결과를 표로 표시한다.
4. ○ 수집 작업은 중단 가능하며 진행률이 표시된다.
5. ○ 세 경로의 결과는 ‘주제 키워드 또는 제목’ 1건으로 수렴한다.
```


### 2.1 경로 A: 키워드 분석 및 목록 일괄 입출력
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: 사용자 | **기기**: Chromium 브라우저 확장(크롬/네이버 웨일/엣지)

메인 키워드로 연관/자동완성 키워드를 수집하고(선택 시 검색량/경쟁도 포함), 표로 표시하며 목록 업로드/다운로드를 지원한다.


#### 2.1.1 연관/자동완성 키워드 수집 및 테이블 표시

메인 키워드를 입력받아 연관/자동완성 키워드를 수집하고 표로 보여준다. (선택) 검색량/경쟁도 컬럼을 표시한다.
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: 사용자 | **기기**: Chromium 브라우저 확장(크롬/네이버 웨일/엣지)


#### 2.1.2 키워드 목록 업로드/다운로드

키워드 목록을 파일로 업로드해 표에 반영하고, 현재 표를 파일로 다운로드한다.
> **중요도**: 🟡 중간 | **진행 상태**: ⚪ 시작전 | **역할**: 사용자 | **기기**: Chromium 브라우저 확장(크롬/네이버 웨일/엣지)


### 2.2 경로 B: 경쟁 블로그 제목 수집
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 시작전 | **역할**: 사용자 | **기기**: Chromium 브라우저 확장(크롬/네이버 웨일/엣지)

대상 블로그의 게시물 제목을 지정 개수만큼 수집해 표로 제공하고, 제목 선택 시 곧바로 본문 생성 입력으로 전달한다.


#### 2.2.1 경쟁 블로그 제목 수집 파이프라인
